IVOA Objects

The classes within this library represent models of metadata within the IVOA. These metadata are mostly to be found in IVOA registries (e.g. the VOResource schema) and thus exclude most of the IVOA "data models" (e.g. The "Spectrum Data Model")

The classes contain two bindings
  • JAXB - for marshalling an unmarshalling to XML defined via the schema to be found at http://www.ivoa.net/xml/
  • JPA - for persisting the objects in a relational database.

Using the XML binding

There are some helper classes for using the xml binding in the {@link net.ivoa.jaxb} package - in particular the {@link net.ivoa.jaxb.IvoaJAXBContextFactory} sets up a JAXB context with all of the packages/schema included that are supported by the library. In addition there are some static utility methods within {@link net.ivoa.jaxb.IvoaJAXBUtils} that can help with the boilerplate code needed with JAXB marshalling an unmarshalling so that unmarshalling can be as easy as

    VOResources resources = IvoaJAXBUtils.unmarshall(this.getClass().getResourceAsStream("/VOResource.xml"), VOResources.class);

Persisting in a Relational Database

The helper classes for persisting to a relational database are in the {@link net.ivoa.jpa} package. There is a {@link net.ivoa.jpa.ResourceJpaController} for storing and fetching {@link net.ivoa.resource.Resource} objects.
